        1. Разбор каждого требования на: актор, команда, событие, query

1. Таск-трекер
Актор - любой аккаунт
Команда - Создать аккаунт, Создать задачу, Заассайнить задачи, Отметить задачу выполненной
Событие - Задача создана (CUD - Create), задача реассайнена (CUD), задача закрыта (CUD), Задача создана
Данные - Аккаунт id (за кем задача), id Задачи, другие поля задачи (описание, статус, )

2. Аккаунтинг
Актор - аккаунт "админ" или "бухгалтер", событие "задача закрыта", "задачи перетассованы", Cron?
Команда - Провести
Событие - баланс изменён
Данные - Аккаунт id, баланс, БалансChange (id задачи, изменение баланса, причина - переассайн или выполнение)

3. Аналитика
Актор - событие "баланс изменён"
Команда - Обновить баланс (CUD)
Событие - баланс изменён (аналитика)
Данные - Аккаунт id, баланс
